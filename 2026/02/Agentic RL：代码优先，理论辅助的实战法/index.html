еҸ‘иЎЁдәҺ2026-02-26|з®—жі•и§ЈжһҗејәеҢ–еӯҰд№ 
|жҖ»еӯ—ж•°:876|йҳ…иҜ»ж—¶й•ҝ:3еҲҶй’ҹ

еҰӮжһңдҪ зҡ„зӣ®ж ҮжҳҜвҖңе°Ҫеҝ«иғҪж”№и®ӯз»ғд»Јз ҒвҖқпјҢжңҖжңүж•Ҳзҡ„и·Ҝеҫ„дёҚжҳҜе…Ҳе•ғе…¬ејҸпјҢиҖҢжҳҜпјҡ

  1. е…Ҳи·‘йҖҡдёҖдёӘжңҖе°ҸеҸҜиҝҗиЎҢе®һзҺ°гҖӮ
  2. зӣҙжҺҘж”№ reward еҸӮж•°пјҢи§ӮеҜҹзӯ–з•ҘиЎҢдёәеҸҳеҢ–гҖӮ
  3. еҶҚеӣһеӨҙз”Ёе°‘йҮҸзҗҶи®әи§ЈйҮҠдҪ зңӢеҲ°зҡ„зҺ°иұЎгҖӮ

иҝҷзҜҮе°ұжҳҜиҝҷдёӘжҖқи·Ҝзҡ„е®һжҲҳзүҲжң¬гҖӮ

1. е…Ҳи·‘йҖҡпјҡжңҖе°Ҹ Agentic RL и„ҡжң¬

жҲ‘е·Із»ҸеңЁд»“еә“йҮҢж”ҫдәҶдёҖдёӘеҸҜиҝҗиЎҢи„ҡжң¬пјҡ

  • /Users/wangpeng/еҚҡе®ў/Hexo_Blog_Source/tools/agentic_rl_code_first.py

е®ғжҳҜдёҖдёӘ code-first зҡ„ Agentic еҶізӯ–е®һйӘҢпјҲcontextual banditпјүпјҡ

  • ANSWER_NOWпјҡзӣҙжҺҘеӣһзӯ”
  • USE_TOOLпјҡи°ғз”Ёе·Ҙе…·еҶҚеӣһзӯ”
  • THINK_MOREпјҡе…ҲжҖқиҖғеҶҚеӣһзӯ”

е®Ңж•ҙж•°жҚ®жөҒйқһеёёжё…жҷ°пјҡ

  • Environment з»ҷзҠ¶жҖҒпјҲconfidence / tool_hint / complexityпјү
  • PolicyNet иҫ“еҮәеҠЁдҪңеҲҶеёғ
  • REINFORCE loss жӣҙж–°зӯ–з•Ҙ

зӣҙжҺҘиҝҗиЎҢпјҡ

2. е…ій”®д»Јз ҒпјҲеҸӘзңӢжңҖж ёеҝғдёүж®өпјү

2.1 зҺҜеўғпјҡеҠЁдҪң -> жҲҗеҠҹжҰӮзҺҮ -> еҘ–еҠұ

дҪ иҰҒзңӢзҡ„йҮҚзӮ№пјҡеҘ–еҠұеҮҪж•°жүҚжҳҜиЎҢдёәжқ жқҶгҖӮ

2.2 зӯ–з•ҘзҪ‘з»ңпјҡзҠ¶жҖҒеҲ°еҠЁдҪңеҲҶеёғ

дҪ иҰҒзңӢзҡ„йҮҚзӮ№пјҡе…ҲеҲ«зә з»“зҪ‘з»ңз»“жһ„пјҢе…ҲзңӢиҫ“е…Ҙ/иҫ“еҮәиҜӯд№үгҖӮ

2.3 и®ӯз»ғпјҡREINFORCE дёҖжӯҘй—ӯзҺҜ

дҪ иҰҒзңӢзҡ„йҮҚзӮ№пјҡreward -> advantage -> loss иҝҷдёҖжқЎй“ҫгҖӮ

3. е…Ҳи·‘пјҢеҶҚж”№пјҡдёӨдёӘжңҖжңүж•Ҳе®һйӘҢ

е®һйӘҢ AпјҡжҸҗй«ҳе·Ҙе…·жҲҗжң¬пјҢи§ӮеҜҹзӯ–з•ҘжҳҜеҗҰеҮҸе°‘вҖңе·Ҙе…·дҫқиө–вҖқ

дҪ дјҡзңӢеҲ° think_ratio дёҠеҚҮпјҢavg_reward дёӢйҷҚпјҲе·Ҙе…·жӣҙиҙөдәҶпјүгҖӮ

е®һйӘҢ Bпјҡз»ҷвҖңжҖқиҖғвҖқжӯЈеҘ–еҠұпјҢи§ӮеҜҹжҳҜеҗҰжӣҙеҖҫеҗ‘е…ҲжҖқиҖғ

дҪ дјҡзңӢеҲ° think_ratio жҳҺжҳҫдёҠеҚҮгҖӮ

иҝҷе°ұжҳҜвҖңе…Ҳи·‘йҖҡпјҢеҶҚжӢҶи§ЈвҖқжңҖж ёеҝғзҡ„еҸҚйҰҲеӣһи·Ҝпјҡ

  • ж”№дёҖдёӘ reward зі»ж•°
  • зңӢеҠЁдҪңеҲҶеёғжҖҺд№ҲеҸҳ
  • еҶҚи§ЈйҮҠдёәд»Җд№ҲеҸҳ

4. CleanRL зҡ„йҳ…иҜ»ж–№ејҸпјҲе·ҘзЁӢеёҲзүҲпјү

дҪ зҡ„зӣ®ж ҮдёҚжҳҜвҖңиҜ»е®ҢжүҖжңүз®—жі•вҖқпјҢиҖҢжҳҜиҜ»жҮӮдёҖдёӘеҚ•ж–Үд»¶е®һзҺ°зҡ„е®Ңж•ҙж•°жҚ®жөҒгҖӮ

жҺЁиҚҗйЎәеәҸпјҡ

  1. е…ҲзңӢ cleanrl/ppo.pyпјҲеҚ•ж–Үд»¶пјҢдё»е№Іе®Ңж•ҙпјү
  2. еҸӘиҝҪиҝҷеҮ иЎҢпјҡcollect rollout -> compute advantage -> policy loss -> optimizer.step
  3. еҜ№з…§дҪ иҮӘе·ұзҡ„жңҖе°Ҹи„ҡжң¬пјҢжүҫвҖңеҗҢжһ„дҪҚзҪ®вҖқ

е»әи®®еҒҡжі•пјҡ

  1. е…Ҳи·‘й»ҳи®ӨеҸӮж•°пјҢзЎ®и®ӨеҸҜеӨҚзҺ°гҖӮ
  2. еҸӘж”№дёҖдёӘеҸҳйҮҸпјҲдҫӢеҰӮ ent_coef жҲ– clip_coefпјүгҖӮ
  3. еҸӘзңӢдёүзұ»ж—Ҙеҝ—пјҡreward / entropy / KL(or clip frac)гҖӮ

5. зҗҶи®әеҸӘиЎҘдёүд»¶дәӢпјҲеӨҹз”ЁдәҶпјү

  1. log pi(a|s)пјҡеӣ дёәзӯ–з•ҘжўҜеәҰиҰҒеҜ№жҰӮзҺҮеҲҶеёғжұӮеҜјгҖӮ
  2. advantageпјҡе‘ҠиҜүжЁЎеһӢвҖңиҝҷж¬ЎеҠЁдҪңжҜ”е№іеқҮеҘҪеӨҡе°‘вҖқгҖӮ
  3. entropyпјҡйҳІжӯўзӯ–з•ҘиҝҮж—©еЎҢзј©еҲ°еҚ•дёҖеҠЁдҪңгҖӮ

е…¶дҪҷзҗҶи®әпјҢзӯүдҪ йҒҮеҲ°е…·дҪ“и®ӯз»ғй—®йўҳеҶҚиЎҘгҖӮ

  2. еңЁеҗҢдёҖи„ҡжң¬йҮҢеўһеҠ  reward breakdown жү“еҚ°пјҲsuccess/cost еҗ„иҮӘеқҮеҖјпјүпјҢйҒҝе…ҚвҖңеҸӘзңӢжҖ» rewardвҖқиҜҜеҲӨгҖӮ
  3. еҶҚжҠҠиҝҷеҘ— reward и®ҫи®ЎиҝҒз§»еҲ°дҪ е®һйҷ…з”Ёзҡ„ Agentic RL жЎҶжһ¶пјҲveRL / RLHF pipelineпјүгҖӮ
